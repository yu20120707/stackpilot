# 发布后异常排查 SOP

当线程中同时出现“发布”“回滚”“错误率上升”“5xx”“报警原因”这类关键词时，优先按下面顺序排查：

1. 确认异常开始时间是否与最近一次发布窗口重合。
2. 对比发布前后错误率、成功率、p95/p99 延迟和核心业务指标。
3. 检查是否只影响新版本实例、灰度实例或某个机房。
4. 如果已经执行回滚，继续观察回滚后错误率、timeout、5xx 是否持续下降。
5. 对照本次变更内容，重点核查配置项、数据库访问、缓存键、下游接口和鉴权逻辑。

经验规则：

- “回滚后错误率下降”通常说明发布变更高度相关，但仍然需要日志和指标作为证据。
- 如果回滚无明显效果，应立即转向下游依赖、数据库连接池、网络抖动或流量突增排查。
- 当影响范围不明确时，摘要里必须保留“不确定性”，不能直接写成最终结论。
